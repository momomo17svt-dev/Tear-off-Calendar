# 日めくりカレンダーアプリ 要件定義・開発フロー統合資料

## 1. アプリ概要とコア要件
写真（画像）を背景に設定でき、毎日の予定や誕生日を確認できるオンデバイス完結型のモダンな日めくりカレンダーアプリ。

### 1.1 機能要件
* **日めくり画面:** 今日の日付、曜日、登録された予定・誕生日をリスト表示。スワイプ操作での日付移動。
* **予定・誕生日管理:** 日付、タイトル、カテゴリ（予定/誕生日）の登録・編集・削除。
* **背景画像設定:** 端末のアルバムから画像を選択し、背景として設定可能。設定画面で表示のON/OFF切り替え。
* **オフライン動作:** ネットワーク通信に依存せず、すべてのデータを端末内に保存。

## 2. UI/UX・画面遷移設計（LINE風ボトムタブ）
モダンで「写真が主役」となるデザインを実現するため、グラスモーフィズム（すりガラス風）と透過UIを活用する。

### 2.1 画面構成（Bottom Tab Navigation）
* **[左タブ] ホーム（日めくりカレンダー）**
  * 背景：ユーザー設定画像（フルスクリーン）。マルチーズやトイプードルなどの写真が映えるよう、UIは透過ベース。
  * 情報パネル：半透明のブラー（ぼかし）カード上に日付と予定リストを表示。
* **[中央タブ] 追加（予定・誕生日の入力）**
  * タブを切り替えるのではなく、タップ時に下から入力画面（モーダル）がせり上がる直感的なUI。
* **[右タブ] 設定**
  * 背景画像ON/OFFトグル、画像アップロードボタン。

※**ボトムタブ自体のデザイン:** 背景画像が画面下部で途切れないよう、タブバーの背景も半透明または完全透過にし、アイコンが浮き上がるスタイルとする。

## 3. システム構成・技術スタック
Windows + VS Code環境での開発を主軸とし、iOSビルドはクラウドに委譲する。

* **開発エディタ:** VS Code
* **フレームワーク:** React Native (Expo)
* **データベース:** Expo SQLite (オンデバイス)
* **状態管理:** Zustand または React Context
* **CI/CD（ビルド・デプロイ）:** GitHub Actions または Codemagic（リポジトリのPushを検知し、クラウドのMac環境で`.ipa` / `.aab`を自動生成）

## 4. データベース設計（拡張対応版）
将来的な機能追加（毎年の繰り返し、色分け、通知）を見据えたテーブル構成。
画像のバイナリはDBに保存せず、`expo-file-system`でアプリ内領域に保存し、URIのみを管理する。

### 4.1 `events` テーブル（予定・誕生日）
| カラム名 | データ型 | 制約 | 説明 |
| :--- | :--- | :--- | :--- |
| `id` | INTEGER | PRIMARY KEY AUTOINCREMENT | 一意の識別子 |
| `title` | TEXT | NOT NULL | 予定のタイトル |
| `date` | TEXT | NOT NULL | 開催日（`YYYY-MM-DD`） |
| `type` | TEXT | NOT NULL | `'birthday'` または `'schedule'` |
| `memo` | TEXT | - | 補足情報 |
| `is_annual` | INTEGER | DEFAULT 0 | 拡張用: 毎年の繰り返しフラグ（0:False, 1:True） |
| `color_code`| TEXT | - | 拡張用: イベントのラベルカラー（Hex値など） |
| `notify_time`| TEXT | - | 拡張用: リマインダー通知時間（`HH:MM`） |
| `created_at`| TEXT | DEFAULT CURRENT_TIMESTAMP | 作成日時 |

### 4.2 `settings` テーブル（アプリ設定）
| カラム名 | データ型 | 制約 | 説明 |
| :--- | :--- | :--- | :--- |
| `key` | TEXT | PRIMARY KEY | 設定項目（`is_bg_enabled`, `bg_uri` 等） |
| `value` | TEXT | - | 設定値（文字列として保存） |
